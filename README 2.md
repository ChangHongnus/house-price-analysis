# House Price Analysis

A end-to-end machine learning project to predict house prices using feature engineering, multicollinearity analysis, and regularized regression and XGBoost models.

## Quickstart

### Step 1: Clone the repository

```bash
git clone https://github.com/YOUR_USERNAME/house-price-analysis.git
cd house-price-analysis
```

### Step 2: Create and activate a virtual environment

**macOS / Linux:**
```bash
python3.12 -m venv venv
source venv/bin/activate
```

**Windows:**
```bash
python3.12 -m venv venv
venv\Scripts\activate
```

> **Note:** If `python3.12` is not found, install it from [python.org](https://www.python.org/downloads/) or via `brew install python@3.12` on macOS.

### Step 3: Upgrade pip and install dependencies

**macOS / Linux:**
```bash
python3.12 -m pip install --upgrade pip setuptools wheel build
pip install -r requirements.txt
```

**macOS Apple Silicon (M1/M2/M3) only** — set these flags before installing if you encounter build errors:
```bash
export ARCHFLAGS="-arch arm64"
export CPPFLAGS="-I/opt/homebrew/include"
export LDFLAGS="-L/opt/homebrew/lib"
pip install -r requirements.txt
```

**Windows:**
```bash
python -m pip install --upgrade pip setuptools wheel build
pip install -r requirements.txt
```

### Step 4: Configure environment variables

Create a `.env` file in the project root:

```
DATA_SOURCE_PATH=../data/raw/
DATA_PROCESSED_PATH=../data/processed/
HOUSE_DATASET_FILENAME=house_dataset.csv
CLEANED_DATA_FILENAME=cleaned_house_data.csv
X_TRAIN_DATA_FILENAME=X_train.csv
X_TEST_DATA_FILENAME=X_test.csv
Y_TRAIN_LOG_DATA_FILENAME=y_train_log.csv
Y_TEST_LOG_DATA_FILENAME=y_test_log.csv
Y_TRAIN_ACTUAL_DATA_FILENAME=y_train_actual.csv
Y_TEST_ACTUAL_DATA_FILENAME=y_test_actual.csv
SOURCE_PATH=../src/
```

### Step 5: Add the raw data

Download `house_dataset.csv` and place it in `data/raw/`.

### Step 6: Run the notebooks in order

Launch Jupyter from within your activated virtual environment:

```bash
jupyter notebook
```

Then run the notebooks in sequence:

| Notebook | Description |
|----------|-------------|
| `01-data-exploration.ipynb` | Exploratory data analysis and distributions |
| `02-data-cleaning.ipynb` | Missing values, outliers, type fixing |
| `03-feature-engineering.ipynb` | Feature creation, VIF analysis, encoding |
| `04-modeling.ipynb` | OLS, Ridge, Lasso, ElasticNet, XGBoost |

> Each notebook saves its outputs to `data/processed/` for the next notebook to load.

## Troubleshooting

**`openpyxl` not found after installing:**
Make sure you installed into the correct virtual environment:
```bash
# Check which Python Jupyter is using
import sys
print(sys.executable)

# Install directly into that Python
/path/from/above -m pip install openpyxl
```

**`FileNotFoundError` for processed data files:**
The processed files are generated by running earlier notebooks. Run notebooks `01` through `03` before `04`.

**Apple Silicon build errors:**
Set the compiler flags in Step 3 before running `pip install -r requirements.txt`.

**`externally-managed-environment` error on macOS:**
This means you are not inside the virtual environment. Activate it first:
```bash
source venv/bin/activate
```
